Simple NI-USRP Streaming

The Simple NI-USRP Streaming sample project demonstrates the following tasks:

  • Streaming data from the input port of the NI 294x/295x device to the host VI
  • Streaming data from the host VI to the output port of the NI 294x/295x device

NI 294x/295x Connectors

Throughout this document, the term input port refers to the RX1 or RX2 connector and output port refers to the TX1 connector.

System Requirements

  • LabVIEW 2013 (32-Bit) Full or Professional Development System.
  • NI-USRP RIO Instrument Design Libraries.
  • (Optional) LabVIEW FPGA Module 2013 to view, open, and edit the included FPGA VI. If you do not have the LabVIEW FPGA Module installed, you can still run this sample project; however, the Project Explorer window shows only the host VIs.

Features

  • Configures an input and output data stream to and from the host, other FPGAs, or inside the FPGA itself.
  • Applies frequency shifting and decimation on the FPGA to reduce data sent to the host, which can prevent buffer overflow.
  • Supports both continuous and finite acquisitions and generations.
  • Provides synchronization for deterministic response times and MIMO systems.

Overview

This sample project contains six host VIs, which run on the host computer, and one FPGA VI, which runs on the FPGA. Each host VI configures the device input and output ports and configures a data stream between the host VI and the NI 294x/295x.

The sample project performs the following tasks:

  • Streaming data from the host to the device
    1. The host VI writes data to the host-to-target DMA FIFO. You can see this FIFO in the Project Explorer window at My Computer»<Device Name>»FIFOs»Tx Streaming (Host).vi.
    2. The FPGA VI reads from this DMA FIFO, processes the data, and sends it to the output port.
  • Streaming data from the device to the host
    1. The FPGA VI acquires data from the input port, processes the data, and writes it to the target-to-host DMA FIFO. You can see this FIFO in the Project Explorer window at My Computer»<Device Name>»FIFOs»Rx Streaming (Host).vi.
    2. The host VI reads from the target-to-host DMA FIFO and displays the data on the host VI front panel.

The output stream uses a Start Trigger to begin generating data to the output port. After the stream starts, the host streams data to the FPGA until you stop the application. If the host does not write data to the FPGA quickly enough, the DMA FIFO does not have enough data to be read, and the FPGA reports a buffer underflow.

The input stream uses a Start Trigger to begin streaming data acquired from the input port. This stream can operate in the following modes:

  • Continuous—The FPGA streams data to the host until you stop the application. If the host cannot accept data quickly enough, the DMA FIFO fills too quickly, and the FPGA reports a buffer overflow.
  • Finite—The FPGA periodically sends a block of samples to the host. Specify the size of each block in the Number of Samples host VI front panel control. A block of data is sent when the Start Trigger is asserted.

    Enable finite mode by clicking Number of Samples is Finite on the host VI front panel.

Host VIs

You can view the host VIs in the Project Explorer window by expanding My Computer. The following host VIs are included:

  • Rx Streaming (Host).vi—Streams data from an input port to the host VI.
  • Tx Streaming (Host).vi—Streams data from the host VI to an output port.
  • Rx Streaming Multi-Channel (Host).vi—Streams data from multiple input ports to the host VI.
  • Full Duplex Streaming (Host).vi—Streams data from the input port to the host VI and from the host VI to the output port.
  • MIMO Rx Streaming Multi-Channel (Host).vi—Synchronizes the Start Triggers of multiple NI 294x/295x devices connected to the same host and synchronously streams data from the host VI to the output ports of the devices. Synchronization helps you correlate data across all devices, which is required for RF MIMO applications. This correlation is typically performed by ensuring each device starts acquiring and/or generating data at the same time. This host VI uses PPS TRIG IN and PPS TRIG OUT to synchronize the Start Triggers across all devices. You can measure synchronization by measuring the acquired signal power level of each device.
  • MIMO Tx Streaming Multi-Channel (Host).vi—Synchronizes the Start Triggers of multiple NI 294x/295x devices connected to the same host and synchronously streams data from the input ports of the devices to the host VI. Synchronization helps you correlate data across all devices, which is required for RF MIMO applications. This correlation is typically performed by ensuring each device starts acquiring and/or generating data at the same time. This host VI uses PPS TRIG IN and PPS TRIG OUT to synchronize the Start Triggers across all devices. You can measure synchronization by measuring the acquired signal power level of each device.

    • For more information about using and connecting multiple NI 294x/295x devices, refer to the NI-USRP Help or to the appropriate specifications for your device:

    • NI 2940R Specifications
    • NI 2942R Specifications
    • NI 2943R Specifications
    • NI 2950R Specifications
    • NI 2952R Specifications
    • NI 2953R Specifications

FPGA VIs

You can view the FPGA VI in the Project Explorer window by expanding My Computer. The following FPGA VI is included:

  • Streaming Xcvr (FPGA).vi—Enables sample generation using the Tx Streaming Engine and sample acquisition using the Rx Streaming Engine. Specify the number of samples to generate using the tx.samples to generate parameter. Specify the number of samples to acquire using the rx.samples to acquire parameter.

If you have NI-USRP RIO Support installed, you can open the FPGA VI by navigating to My Computer»<FPGA VIs>»FPGA name.vi. You can modify the FPGA VI or create one as needed for your application.

Running this Sample Project

  1. Ensure all hardware is set up and visible in MAX. Record the resource names of all NI 294x/295x devices; you will need these names later.
  2. If you are using a non-MIMO host VI configuration:
    1. In the Project Explorer window, open a non-MIMO host VI, such as Rx Streaming (Host).vi or Tx Streaming (Host).vi.
    2. In the RIO Device pull-down menu, specify the resource name of the NI 294x/295x device that you identified in Step 1.

    If you are using a host MIMO VI configuration:
    1. Wire and configure all your devices to use the same Reference Clock. Configuring your devices to use the same Reference Clock ensures that the clocks and triggers are synchronized.
    2. Wire the PPS TRIG OUT from one device, the master, to the PPS TRIG IN of all the devices. An Ettus Research OctoClock is the recommended solution to connect the PPS TRIG lines.
    3. In the Project Explorer window, open a MIMO host VI, such as MIMO Rx Streaming Multi-Channel (Host).vi or MIMO Tx Streaming Multi-Channel (Host).vi.
    4. Update the RIO Devices array with the resource names of each NI 294x/295x device. You identified these resource names in Step 1. The master device must be the first element in the array. Slave devices follow.
  3. Click Run. The front panel displays the acquired data.
  4. Click Stop.
  5. Experiment with different values for the front panel controls and click Run. You must restart the VI for new settings to take effect.
